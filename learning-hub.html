Portfolio Repository Structure

Every role-based portfolio follows this exact structure:

Senior-Manager-SRE/                  # Repo name
├── index.html                       # Main portfolio page
├── style.css                        # External stylesheet
├── script.js                        # JavaScript (dark mode)
├── favicon.svg                      # Browser tab icon
├── Rajesh_Dammala_Resume.pdf       # Downloadable resume
├── README.md                        # Tech stack ONLY (4-8 lines)
└── .gitignore                       # Git tracking rules

Critical Rules for Portfolios

  • ✅ index.html — Single-page portfolio (no other HTML files)
  • ✅ style.css — External CSS file (NOT inline <style> tags)
  • ✅ script.js — Vanilla JavaScript (no frameworks like React)
  • ✅ favicon.svg — SVG format (scalable, theme colors)
  • ✅ .pdf file — Git-tracked (remove *.pdf from .gitignore)
  • ✅ .gitignore — Only excludes *.docx, .DS_Store, node_modules/

Why External CSS/JS?

  • 🚀 GitHub Pages caches them efficiently
  • 📝 Cleaner HTML structure
  • 🔧 Easier to maintain and update
  • ⚡ Browser can render while loading

Question 3: "Show me a PowerShell script you've written"

Demo Script (Sync-GitHub-Repos.ps1):

param(
    [string]$SyncType = "all",
    [switch]$DryRun
)

$repos = @(
    @{name="personal_rdammala"; url="https://github.com/rdammala/personal_rdammala"},
    @{name="Senior-Manager-SRE"; url="https://github.com/rdammala/Senior-Manager-SRE"}
    # ... 25 more repos
)

foreach ($repo in $repos) {
    Write-Host "Processing: $($repo.name)"
    
    try {
        if (-not (Test-Path $repo.name)) {
            if (-not $DryRun) {
                git clone $repo.url
            }
            Write-Host "  [NEW] Cloned repository"
        }
        else {
            if (-not $DryRun) {
                git -C $repo.name fetch origin
                git -C $repo.name pull origin main
            }
            Write-Host "  [SYNCED] Updated from remote"
        }
    }
    catch {
        Write-Host "  [ERROR] $_" -ForegroundColor Red
    }
}

Write-Host "Sync complete!" -ForegroundColor Green

Key Points to Highlight:

  • Error handling with try-catch
  • Parameterization (SyncType, DryRun)
  • Idempotent design (safe to run multiple times)
  • Clear status messages
  • Git command usage with -C flag
  • Array iteration

Question 4: "How do you handle errors in your scripts?"

Error Handling Strategies:

1. Try-Catch Blocks:

try {
    git -C $repo.name pull origin main
}
catch {
    Write-Host "Error pulling $($repo.name): $_"
    $errors++
}

2. Exit Code Checking:

if ($LASTEXITCODE -ne 0) {
    Write-Host "Git command failed"
}

3. Input Validation:

if (-not (Test-Path $path)) {
    Write-Host "Directory not found"
    return
}

4. Dry-Run Mode: Preview changes before executing

5. Logging: Log all operations with timestamps
