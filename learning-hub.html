What is This Project?

The personal_rdammala project is a complete automation system demonstrating:

  • ✅ Version control with Git & GitHub (27 repos)
  • ✅ PowerShell automation for repository synchronization
  • ✅ Resume generation with Node.js templating
  • ✅ HTML/CSS/JS portfolio websites deployed to GitHub Pages
  • ✅ Infrastructure as Code using Bicep
  • ✅ DevOps best practices and learning materials

Why Build This?

🎯 Problem: Manual Resume Versioning

Creating separate resumes for different companies wastes time. Solution: Automate with templates.

🎯 Problem: Portfolio Scattered Across Repos

27 GitHub repos hard to sync. Solution: PowerShell script syncs all in <2 minutes.

🎯 Problem: Interview Prep Material Lost

DevOps knowledge scattered. Solution: 88 organized learning documents covering Git, CI-CD, Bicep.

Git Commands Cheat Sheet

🔧 Essential Git Commands

git clone <URL>              # Clone repository
git add .                     # Stage all changes
git commit -m "message"       # Commit changes
git push origin main          # Push to remote
git pull origin main          # Fetch and merge
git branch -a                 # List all branches
git checkout -b feature/name  # Create new branch
git merge feature/name        # Merge branch to main
git log --oneline             # Show commit history
git status                    # Show repo status

PowerShell Scripting Basics

🔧 Common PowerShell Patterns

# Variables
$repos = @("repo1", "repo2")

# Loops
foreach ($repo in $repos) {
    Write-Host "Syncing $repo"
    git -C $repo pull
}

# Error handling
try {
    git clone $url
} catch {
    Write-Host "Error: $_"
}

# String interpolation
$message = "Hello $name"

9️⃣ Interview Guide

Question 1: "Tell me about a project where you used Git"

Your Answer (2-3 minutes):

"I built an automation system managing 27 GitHub repositories using Git. The project includes:

  • PowerShell sync script that clones, fetches, and pulls updates from all 27 repos
  • 9 role-based portfolio repositories deployed to GitHub Pages
  • Monorepo pattern with learning materials, scripts, and infrastructure code

Key Git skills demonstrated:

  • Branching strategy (main + feature branches)
  • Commit hygiene (atomic commits, clear messages)
  • Merge conflict resolution
  • Idempotent sync operations (safe to run multiple times)

Question 2: "Explain your PowerShell automation script"

Your Answer (2-3 minutes):

"My Sync-GitHub-Repos.ps1 script demonstrates error handling and batch processing:

foreach ($repo in $repositories) {
    # Clone if missing
    if (-not (Test-Path $repo.path)) {
        git clone $repo.url
    }
    # Pull if exists
    else {
        git -C $repo.path fetch origin
        git -C $repo.path pull origin main
    }
}

Key features:

  • Idempotent design (safe to run multiple times)
  • Error handling with try-catch
  • Dry-run mode for preview
  • Status reporting

Question 3: "How do you approach documentation?"

Your Answer (1-2 minutes):

"I practice 'documentation as code':

  • Clear folder structure with numbered modules (0-PROJECT-OVERVIEW.md)
  • Code examples with syntax highlighting
  • README.md at each level (root, components, portfolios)
  • Automated diagram generation (Mermaid)

This project includes 88 learning documents explaining Git, CI-CD, Bicep, and DevOps concepts."
