- **Web Development:** Static HTML/CSS/JS portfolios with Dark Mode

- **Documentation as Code:** Markdown-driven content structure
- **Resume/Portfolio Generation:** Templating, versioning, deployment

## 💼 Interview Talking Points

### **Question: "Tell me about a project where you used Git and GitHub?"**

**Your Answer Structure:**
1. **Problem:** Managing 27 GitHub repos (portfolios, learning materials, tools) locally
2. **Solution:** Built PowerShell sync script with git clone, fetch, pull, merge
3. **Technologies:** Git, GitHub, PowerShell, Bash, YAML
4. **Challenges:** Handling merge conflicts, managing branches, dry-run mode, error recovery
5. **Results:** Automated sync of 27 repos in <2 minutes, reduced manual work by 95%

### **Question: "How do you structure a DevOps project?"**

**Your Answer:**
- **Monorepo pattern** for related projects (personal_rdammala)
- **Branching strategy:** Master + feature branches per role
- **Infrastructure as Code:** Bicep templates in dedicated folders
- **Documentation:** Markdown-driven (this Learning Hub)
- **Automation:** PowerShell scripts for repetitive tasks
- **CI-CD:** GitHub Actions (planned) for deployment

### **Question: "Explain your resume/portfolio automation?"**

**Your Answer:**
- Node.js generator creates .docx from templates (tailored to JD keywords)
- GitHub Pages deployment for role-based portfolios
- Markdown files track application status
- Single resume generator → multiple tailored outputs
- Versioning system (v1, v2, v3) for tracking updates
